# PRD: มะหฺรอมเช็ค (Mahram Check)

## 1. ภาพรวมผลิตภัณฑ์

### 1.1 ชื่อผลิตภัณฑ์
**มะหฺรอมเช็ค** (فحص المحرم) — Mahram Check LIFF App

### 1.2 ปัญหาที่ต้องการแก้
มุสลิมจำนวนมากไม่แน่ใจว่าบุคคลที่มีความสัมพันธ์กับตนเป็น **มะหฺรอม** (محرم) หรือไม่ — คือบุคคลที่อนุญาตให้สัมผัส (จับมือสลาม) ได้ตามหลักศาสนาอิสลาม กฎเกณฑ์มีความซับซ้อนเนื่องจากครอบคลุม 3 ประเภท (สายเลือด, สายนม, สายสมรส) และมีข้อยกเว้นหลายกรณี

### 1.3 วิธีแก้ปัญหา
แอปพลิเคชันบน LINE LIFF ที่ให้ผู้ใช้พิมพ์ความสัมพันธ์ (เช่น "ลุง", "พี่ชายของสามี") แล้วระบบตอบทันทีว่าเป็นมะหฺรอมหรือไม่ พร้อมเหตุผลอ้างอิงตามหลักฟิกฮ์

### 1.4 กลุ่มเป้าหมาย
- มุสลิมในประเทศไทยที่ใช้ LINE เป็นหลัก
- ทุกเพศ ทุกวัย ที่ต้องการตรวจสอบความสัมพันธ์มะหฺรอม
- ผู้ที่เพิ่งเข้ารับอิสลามและยังไม่คุ้นเคยกับกฎเกณฑ์

### 1.5 เป้าหมายหลัก
| เป้าหมาย | ตัวชี้วัด |
|-----------|----------|
| ตอบคำถามได้ทันที | Rule-based ครอบคลุม ≥ 90% ของคำถามทั่วไป |
| ใช้งานง่าย | ตอบได้ภายใน 2 แตะ (เลือกเพศ → พิมพ์/กดตัวอย่าง) |
| เข้าถึงง่าย | เปิดผ่าน LINE ได้เลย ไม่ต้องติดตั้งแอปเพิ่ม |
| ค่าใช้จ่ายเป็นศูนย์ | ใช้ free tier ทุกบริการ (Vercel, Gemini, LINE) |

## 3. ข้อกำหนดด้านข้อมูล (Data Requirements)

### 3.1 ฐานข้อมูลมะหฺรอม (Rule Engine)

#### สำหรับผู้หญิง (FDB) — ตรวจสอบผู้ชายที่เป็นมะหฺรอม
| หมวด | จำนวนรายการ | ตัวอย่าง |
|------|------------|---------|
| สายเลือด (نسب) | ~20 | พ่อ, ปู่/ตา, ลูกชาย, พี่/น้องชาย, ลุง, อา, น้าชาย, หลานชาย |
| สายสมรส (مصاهرة) | ~5 | สามี, พ่อสามี, ปู่สามี, ลูกเลี้ยงชาย, พ่อเลี้ยง |
| สายนม (رضاعة) | ~2 | พ่อนม, พี่น้องชายร่วมนม |
| ไม่ใช่มะหฺรอม | ~8 | ลูกพี่ลูกน้อง, พี่/น้องเขย, พี่/น้องชายสามี, เพื่อน, แฟน |

#### สำหรับผู้ชาย (MDB) — ตรวจสอบผู้หญิงที่เป็นมะหฺรอม
| หมวด | จำนวนรายการ | ตัวอย่าง |
|------|------------|---------|
| สายเลือด (نسب) | ~20 | แม่, ย่า/ยาย, ลูกสาว, พี่/น้องสาว, ป้า, น้าสาว, หลานสาว |
| สายสมรส (مصاهرة) | ~6 | ภรรยา, แม่ยาย, ย่า/ยายภรรยา, ลูกเลี้ยงหญิง, แม่เลี้ยง, ลูกสะใภ้ |
| สายนม (رضاعة) | ~2 | แม่นม, พี่น้องสาวร่วมนม |
| ไม่ใช่มะหฺรอม | ~8 | ลูกพี่ลูกน้อง, พี่/น้องสะใภ้, พี่/น้องสาวภรรยา, เพื่อน, แฟน |

#### Pattern Matching (คำถามซับซ้อน)
- ~40 regex patterns สำหรับความสัมพันธ์หลายชั้น
- ตัวอย่าง: "แม่ของแม่ของสามี", "ลูกชายของพี่สาวของภรรยา"

### 3.2 Text Normalization
- ตัดคำลงท้าย: ค่ะ, คะ, ครับ, นะ, จ้า, หน่อย
- ตัดคำแสดงความเป็นเจ้าของ: ของฉัน, ของเรา, ของผม
- แก้ไขสระ: เเ → แ
- ตัดช่องว่างซ้ำ, lowercase

### 3.3 AI Prompt (Gemini)
- บทบาท: นักวิชาการอิสลามเชี่ยวชาญเรื่องมะหฺรอม
- พิจารณา 3 ประเภท: نسب, رضاعة, مصاهرة
- Output: JSON `{mahram, type, reason}`
- Temperature: 0.1 (เน้นความแม่นยำ)
- Max tokens: 500

## 4. ข้อกำหนดด้านเทคนิค (Technical Requirements)

### 4.1 สถาปัตยกรรม

```
┌──────────────────────────────────┐
│       LINE LIFF App (Client)     │
│    public/index.html (static)    │
│                                  │
│  ┌────────────┐  ┌────────────┐  │
│  │ Rule Engine │  │ AI Fallback│  │
│  │ (in-browser)│  │  (fetch)   │  │
│  └────────────┘  └─────┬──────┘  │
└─────────────────────────┼────────┘
                          │ POST /api/mahram
                          ▼
              ┌─────────────────────┐
              │  Vercel Serverless  │
              │   api/mahram.js     │
              └─────────┬───────────┘
                        │
                        ▼
              Google Gemini 2.0 Flash
```

### 4.2 Tech Stack
| ชั้น | เทคโนโลยี |
|------|----------|
| Frontend | HTML + CSS + Vanilla JS (single file) |
| Backend | Vercel Serverless Function (Node.js) |
| AI | Google Gemini 2.0 Flash API |
| Platform | LINE LIFF SDK v2 |
| Hosting | Vercel (free tier) |
| Font | IBM Plex Sans Thai + Noto Naskh Arabic |

### 4.3 API Endpoint

#### `POST /api/mahram`

**Request:**
```json
{
  "gender": "male" | "female",
  "input": "ความสัมพันธ์ที่ต้องการตรวจสอบ"
}
```

**Response (สำเร็จ):**
```json
{
  "mahram": true | false | null,
  "type": "สายเลือด (نسب)" | "สายนม (رضاعة)" | "สายสมรส (مصاهرة)" | "คู่สมรส" | "ไม่ใช่มะหฺรอม" | "ไม่สามารถระบุได้",
  "reason": "เหตุผลภาษาไทย"
}
```

**Error Codes:**
| Code | สาเหตุ |
|------|--------|
| 400 | ไม่ส่ง gender หรือ input |
| 405 | ใช้ method อื่นที่ไม่ใช่ POST |
| 500 | API key ไม่ได้ตั้งค่า หรือ Gemini API error |

|--------|---------|--------|

## 5. ข้อกำหนดด้าน UX/UI

### 5.1 Flow หลัก

```
เปิดแอป → เลือกเพศ → พิมพ์ / กดตัวอย่าง → กด "เช็ค"
                                                  │
                                    ┌─────────────┴──────────────┐
                                    ▼                            ▼
                              Rule-based                    AI (Gemini)
                              (ตอบทันที)                 (รอ 1-3 วินาที)
                                    │                            │
                                    └─────────────┬──────────────┘
                                                  ▼
                                        แสดงผลลัพธ์ + แชร์
```

### 5.2 หน้าจอ
1. **หน้าเลือกเพศ** — logo, ชื่อแอป, ปุ่มชาย/หญิง
2. **หน้าหลัก** — topbar, ช่องค้นหา, quick tags, ผลลัพธ์, ประวัติ, disclaimer

### 5.3 Design System
- สีหลัก: ทอง `#d4a853` (สื่อถึงอิสลาม)
- สีเขียว (มะหฺรอม): `#3a9e6e`
- สีแดง (ไม่ใช่มะหฺรอม): `#d45555`
- พื้นหลัง: gradient อบอุ่น (`#fdf9f3` → `#e8ddd0`)
- Border radius: 14-24px (rounded, modern)
- Animation: float, slideUp, popIn

### 5.4 Responsive
- Max-width: 480px (mobile-first)
- รองรับ viewport height: `100dvh`
- ปิด user-scalable เพื่อ native feel
